# SummaryStreet Platform

A modern, AI-powered blogging platform built with the latest 2026 tech stack.

## Tech Stack (Latest Versions — April 2026)

| Layer | Technology | Version |
|---|---|---|
| Framework | Next.js | 16.2.4 |
| UI Library | React | 19.2.5 |
| Database + Auth | Supabase | @supabase/supabase-js 2.103.3 |
| SSR Auth | @supabase/ssr | 0.10.2 |
| AI Summaries | Groq SDK | 1.1.2 |
| Language | TypeScript | 5.x |

## Features

- **Role-Based Auth** — Author, Viewer, Admin via Supabase Auth
- **AI Summaries** — ~200-word summaries generated by Groq (`llama-3.3-70b-versatile`) on every new post
- **AI Cover Images** — Groq picks Unsplash search terms and assigns a relevant cover image automatically
- **Blog Posts** — Create, edit, delete with automatic featured image assignment
- **Comments** — Per-post comment threads for authenticated users
- **Search** — Debounced live search across post titles
- **Pagination** — 9 posts per page with smart ellipsis pagination
- **Dashboard** — Authors see own posts, Admins see all + stats

## Project Setup

### 1. Clone and Install

```bash
cd <project-folder>
npm install
```

### 2. Set Up Supabase

1. Go to [supabase.com](https://supabase.com) → Create a new project
2. Go to **SQL Editor** → paste the contents of `supabase/schema.sql` and run it
3. Go to **Settings → API** → copy your URL and anon key

### 3. Get Groq API Key

1. Sign up free at [console.groq.com](https://console.groq.com)
2. Create an API key

### 4. Configure Environment

Edit `.env.local` and fill in:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
GROQ_API_KEY=gsk_your-groq-key
UNSPLASH_ACCESS_KEY=your-unsplash-access-key
```

`UNSPLASH_ACCESS_KEY` is optional. If missing, the app still uses reliable Unsplash image URLs via curated fallbacks.

### 5. Run Locally

```bash
npm run dev
```

Open [http://localhost:3001](http://localhost:3001)

## Database Schema

```sql
users      — id, name, email, role

comments   — id, post_id, user_id, comment_text, created_at
```

## AI Summary Flow

1. Author creates a post → `POST /api/posts`
2. Server calls `lib/groq.ts` with the post title + content
3. Groq returns a ~200-word summary using `llama-3.3-70b-versatile`
4. Summary stored in the `posts.summary` column in Supabase
5. Summary displayed on the home listing page and post detail page

## AI Cover Image Flow

1. Author creates or edits a post
2. Server asks Groq for a short visual query based on title + content
3. App fetches a matching Unsplash image via API when `UNSPLASH_ACCESS_KEY` is configured
4. If API lookup or AI query generation fails, app uses a reliable curated Unsplash fallback image URL

## User Roles

| Role | Permissions |
|---|---|
| Author | Create posts · Edit own posts · Comment |
| Viewer | Read posts · Read summaries · Comment |
| Admin | Edit any post · View all on dashboard · Monitor comments |
